# Baseline Before Optimization

## Principle

Do not optimize a RAG system before you can measure its current behavior.

## Checklist

- Can I run the system end-to-end?
- Do I have representative questions?
- Do I know whether retrieval found the right documents?
- Do I know whether the generated answer contains expected information?
- Do I have failed examples?
- Can I rerun the same evaluation after every change?
